# Архитектура и Особенности Проекта

Summagram — это модульное приложение класса **Agentic RAG** (Retrieval-Augmented Generation), предназначенное для решения проблемы информационной перегрузки в Telegram. Оно позволяет индексировать сообщения, файлы и медиаданные, а затем взаимодействовать с ними с помощью естественного языка.

## Архитектурная Схема

Система разделена на три основных независимых сервиса, взаимодействующих через API:

1.  **Backend (API & Logic)**:
    *   Предоставляет унифицированный API для сессий чата, RAG, и загрузки медиа.
    *   Оркестрирует обращения к базам данных и AI-моделям.
2.  **ETL (Extract, Transform, Load)**:
    *   Занимается сбором данных из внешних источников (Telegram).
    *   Управляет процессом обработки, индексации и извлечения сущностей.
3.  **Model Orchestrator & AI Services**:
    *   **model_orchestrator**: Динамически управляет распределением GPU-ресурсов между специализированными сервисами-инференсами.
    *   **sglang** (Text/Vision): Высокопроизводительные серверы локального инференса.
    *   **whisper**: Локальный сервис транскрибации аудио.
4.  **Frontend (UI)**:
    *   Предоставляет графический интерфейс для пользователя (Next.js).
    *   Управляет сессиями чата и отображением аналитики.

## Технологический Стек

### База Данных и Хранение
-   **SQLite (aiosqlite)**: Используется как основное хранилище метаданных, истории чатов и результатов работы ETL.
    *   *Почему:* Простота развертывания, отсутствие необходимости в отдельном сервере БД, отличная производительность для локальных нагрузок.
-   **ChromaDB**: Векторная база данных для хранения эмбеддингов и семантического поиска.
    *   *Почему:* Легковесность, нативная поддержка LlamaIndex, возможность работы как в режиме библиотеки, так и в режиме клиента.

### ML-модели и Фреймворки
-   **LlamaIndex**: Основной фреймворк для реализации RAG.
    *   *Почему:* Мощные инструменты для структурирования данных и управления контекстом.
-   **FastAPI**: Основа для Backend, ETL и Model Orchestrator сервисов.
    *   *Почему:* Высокая производительность, асинхронность и автоматическая документация API.
-   **Models & Inference Engines**:
    *   **SGLang**: Идеально подходит для высоконагруженного локального инференса (Text: Qwen2.5-Coder-7B-QA-AWQ / Vision: clip-vit-base-patch32).
    *   **Whisper (Faster-Whisper)**: Сервис анализа аудио-файлов.
    *   **Embeddings:** Локальные модели HuggingFace через локальный инференс или OpenAI.

### Frontend
-   **Next.js / React (v0)**: Современный стек для создания быстрых и интерактивных интерфейсов.
-   **Tailwind CSS & Radix UI**: Для обеспечения премиального дизайна и доступности.
-   **Force-Graph & Recharts**: Для визуализации социальных связей (графа чатов) и статистики.

## Особенности реализации
-   **Agentic RAG**: Вместо простого поиска, система использует "агентский" подход — извлечение структурированных данных (событий) и комплексный анализ истории.
-   **Препроцессинг NLP**: Перед отправкой запроса в LLM используется 5-стадийный детерминированный пайплайн (очистка, NER через `natasha` / rules, разрешение морфологии через `pymorphy3` и парсинг дат через `dateparser`). Это снижает галлюцинации и ускоряет классификацию намерений (intent routing).
-   **Privacy-First & Local ML**: Вся обработка и хранение данных, включая инференс LLM (vision, text, audio), происходят локально, что критично для личных переписок.
-   **Модульность и Оркестрация**: Сервисы разворачиваются в независимых Docker-контейнерах. Если GPU один, `model_orchestrator` берет на себя переключение (заморозку/разморозку) тяжелых сервисов (Text, Vision, Audio) "на лету", чтобы предотвратить OOM.
